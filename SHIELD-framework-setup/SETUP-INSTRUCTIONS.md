# SHIELD Scanner - Docker Container Setup (Simplified)

## Overview

This approach is **much simpler** - we build the container image directly in SHIELD-scanner repository. The Dockerfile:
1. Clones the public SHIELD-framework repository
2. Bundles it into the container
3. Adds a wrapper script to consolidate outputs

**No modifications to SHIELD-framework needed!**

## Files in This Folder

1. **`Dockerfile`** - Self-contained container that clones SHIELD-framework
2. **`entrypoint-wrapper.sh`** - Wrapper that runs scans and consolidates outputs
3. **`.github-workflows-publish.yml`** - GitHub Action to build & publish (optional - can also build locally)

## Option A: Build and Publish via GitHub Actions (Recommended)

### Step 1: Add Workflow to SHIELD-scanner

```bash
cd /path/to/SHIELD-scanner

# Copy Dockerfile and wrapper to root
cp SHIELD-framework-setup/Dockerfile ./
cp SHIELD-framework-setup/entrypoint-wrapper.sh ./
chmod +x entrypoint-wrapper.sh

# Copy workflow
mkdir -p .github/workflows
cp SHIELD-framework-setup/.github-workflows-publish.yml .github/workflows/publish-container.yml
```

### Step 2: Update Workflow to Point to SHIELD-scanner

Edit `.github/workflows/publish-container.yml` and change line 11:
```yaml
env:
  REGISTRY: ghcr.io
  IMAGE_NAME: georges034302/shield-framework  # The container name
```

### Step 3: Commit and Push

```bash
git add Dockerfile entrypoint-wrapper.sh .github/workflows/publish-container.yml
git commit -m "Add SHIELD Framework container build"
git push origin main
```

### Step 4: Monitor Build

1. Go to: https://github.com/Georges034302/SHIELD-scanner/actions
2. Watch the "Build and Publish Container" workflow
3. First build takes ~2-3 minutes (cloning framework + installing tools)

### Step 5: Make Package Public

1. Go to https://github.com/Georges034302?tab=packages
2. Click on `shield-framework`
3. Settings → Danger Zone → Change visibility
4. Select "Public"
5. Confirm

## Option B: Build Locally and Push (For Testing)

### Step 1: Build Image Locally

```bash
cd /path/to/SHIELD-scanner

# Copy files if not done yet
cp SHIELD-framework-setup/Dockerfile ./
cp SHIELD-framework-setup/entrypoint-wrapper.sh ./

# Build the image
docker build -t ghcr.io/georges034302/shield-framework:latest .
```

### Step 2: Test the Image

```bash
# Test help
docker run --rm ghcr.io/georges034302/shield-framework:latest --help

# Test a scan (creates output/ directory)
mkdir -p output
docker run --rm \
  -v $(pwd)/output:/app/output \
  ghcr.io/georges034302/shield-framework:latest \
  --mode posture \
  https://example.com
  
# Check outputs
ls -la output/
cat output/report.json
```

### Step 3: Push to GHCR

```bash
# Login to GHCR
echo $GITHUB_TOKEN | docker login ghcr.io -u Georges034302 --password-stdin

# Or use gh CLI
gh auth token | docker login ghcr.io -u $(gh api user --jq .login) --password-stdin

# Push
docker push ghcr.io/georges034302/shield-framework:latest
```

### Step 4: Make Package Public

Same as Option A, Step 5 above.

## How It Works

### Build Time:
1. Dockerfile clones `https://github.com/Georges034302/SHIELD-framework.git` to `/app/framework`
2. Installs system dependencies (curl, jq, openssl, netcat, dnsutils)
3. Copies entrypoint wrapper script
4. Creates `/app/output` directory for results

### Runtime:
1. Container starts with entrypoint wrapper
2. Wrapper calls `/app/framework/scripts/run_all.sh -o /app/output [args]`
3. Framework executes ~69 security checks across 6 steps
4. Each check writes JSON to `output/step*/*.json`
5. Framework's `generate_report.sh` creates `output/report.md` (automatic)
6. Wrapper consolidates all step JSONs into single `output/report.json`
7. Scanner workflow generates `output/report.html` from JSON (later step)

### Output Files Generated:
- **`output/step1/` through `step6/`** - Individual check results (*.json)
- **`output/report.json`** - Consolidated findings with meta (grade, target, mode) + findings array
- **`output/report.md`** - Full markdown report with remediation guidance (generated by framework)
- **`output/report.html`** - Standalone HTML report (generated by Scanner after container completes)

### Volume Mounts:
- `/work` - GitHub workspace (for authorization files if using authorized mode)
- `/app/output` - Results directory (mounted to `output/` in workspace)

## Advantages of This Approach

✅ **No modifications to SHIELD-framework** - It stays pristine
✅ **Self-contained** - Everything in one Dockerfile  
✅ **Always up-to-date** - Clones latest framework code at build time
✅ **Easy to test** - Build locally before pushing
✅ **Single source of truth** - SHIELD-scanner controls the container

## Troubleshooting

**Problem:** Build fails during git clone
- **Solution:** Check that SHIELD-framework repository is public

**Problem:** Build fails with "permission denied"
- **Solution:** Check repository Settings → Actions → Workflow permissions → "Read and write permissions"

**Problem:** Container starts but crashes immediately
- **Solution:** Check that all SHIELD Framework dependencies are installed (review Dockerfile RUN command)

**Problem:** No report.json generated
- **Solution:** Check container logs - wrapper shows each step. Verify framework scripts are executable.

**Problem:** Scan completes but no findings
- **Solution:** Check if target URL is accessible from container. Try with a known-vulnerable test site first.

## Expected Output Structure

After running a scan, the container generates:

```
output/
  ├── step1/
  │   ├── scope.json
  │   ├── wp_version.json
  │   └── ...
  ├── step2/ ...
  ├── step3/ ...
  ├── step4/ ...
  ├── step5/ ...
  ├── step6/ ...
  ├── report.json      ← Consolidated JSON (all findings)
  ├── report.md        ← Markdown report (generated by framework)
  └── report.html      ← HTML report (generated by Scanner pipeline)
```

The Scanner expects:
- **report.json** - JSON with `meta` (grade, target, mode) + `findings` array  
- **report.md** - Full markdown report
- **report.html** - Generated by Scanner from JSON

## Command Line Interface

The container accepts the same arguments as SHIELD Framework's `run_all.sh`:

```bash
docker run --rm \
  -v $(pwd)/output:/app/output \
  ghcr.io/georges034302/shield-framework:latest \
  --mode posture \
  https://example.com
```

**Common arguments:**
- `--mode <posture|authorized>` - Scan mode (default: posture)
- `--i-accept-risk` - Required for authorized mode
- `--authorization-ref <file>` - Path to authorization document
- `--rate-aware` - Enable respectful scanning with delays
- `-t <sec>` - Timeout in seconds per check

**Note:** The container always writes to `/app/output` which should be volume-mounted.
